1. Общие сведения

Тип проекта: 2D пиксельный платформер
Технологии: Python, библиотека Arcade
Формат выполнения: учебный проект (групповой, до 3 человек)

Проект разрабатывается в рамках изучения библиотеки Arcade и направлен на применение игровых технологий, таких как физика, столкновения, тайловые уровни и работа со сценами.

2. Идея проекта

SuperAdventures — это 2D пиксельная игра в жанре платформер.
Игрок управляет персонажем, который перемещается по уровням, преодолевает препятствия, взаимодействует с объектами окружения и собирает игровые предметы.

Игра построена на прохождении нескольких уровней, каждый из которых имеет собственную структуру и набор объектов.

3. Цель проекта

Создание работоспособного игрового приложения с использованием основных возможностей библиотеки Arcade, демонстрирующего навыки работы с физическим движком, тайловыми картами, столкновениями и структурированным кодом.

4. Основной функционал
4.1 Игровой процесс

Игрок управляет персонажем с помощью клавиатуры.
Персонаж может:

перемещаться влево и вправо;

выполнять прыжки;

взаимодействовать с объектами уровня.

Во время игры происходит подсчёт игровых показателей (например, собранных предметов), которые отображаются на экране.

5. Физика и взаимодействие с персонажем

В проекте используется встроенный физический движок библиотеки Arcade —
PhysicsEnginePlatformer.

Физический движок отвечает за:

действие силы тяжести;

корректную обработку прыжков;

движение персонажа по платформам;

столкновения с объектами уровня.

Персонаж реализован в виде спрайта.
Горизонтальное движение осуществляется изменением параметра change_x, прыжок — изменением change_y.

Прыжок возможен только при нахождении персонажа на поверхности, что проверяется с помощью метода can_jump().

Физический движок предотвращает прохождение персонажа сквозь платформы и обеспечивает корректное взаимодействие с границами объектов.

6. Взаимодействие с объектами уровня

Взаимодействие персонажа с объектами реализуется с помощью проверки столкновений (collide) между спрайтами.

При столкновении персонажа с интерактивными объектами:

объекты могут удаляться со сцены;

учитываются игровые действия (например, сбор предметов);

изменяются игровые параметры в процессе прохождения уровня.

Все игровые объекты организованы в группы спрайтов для удобства обработки.

7. Реализация уровней

Игра содержит несколько уровней.
Каждый уровень представляет собой отдельную карту с описанием:

расположения платформ;

интерактивных объектов;

стартовой позиции персонажа.

Уровни загружаются и инициализируются при начале прохождения.

8. Использование Tiled и TMX

Для создания уровней используется редактор Tiled.
Карты создаются в формате TMX.

С помощью Tiled:

визуально размещаются тайлы;

создаются платформы;

добавляются игровые объекты;

задаётся структура уровня.

TMX-файлы загружаются в приложении средствами библиотеки Arcade.
На основе данных карты автоматически создаются спрайты и добавляются в игровую сцену.

Использование TMX позволяет:

изменять уровни без изменения кода;

быстро добавлять новые уровни;

отделить игровую логику от описания карт.

9. Структура уровней

Каждый уровень содержит:

слой с платформами (статические объекты);

слой с интерактивными объектами;

данные, необходимые для корректной загрузки уровня.

При необходимости структура уровней может быть расширена дополнительными слоями.

10. Технические требования

В проекте используются следующие технологии и возможности Arcade:

файл requirements.txt для управления зависимостями;

спрайты;

столкновения (collide);

анимация;

несколько уровней;

камера;

система частиц;

звуковые эффекты;

физический движок;

механизм хранения данных (файл или база данных).

11. Архитектура проекта

Код проекта разделён на логические модули и классы:

управление персонажем;

логика уровней;

обработка физики;

работа со сценой;

пользовательский интерфейс.

Такой подход упрощает поддержку и дальнейшее развитие проекта.

12. Требования к качеству кода

соблюдение рекомендаций PEP 8;

использование констант;

говорящие имена переменных и методов;

минимизация дублирования кода;

применение объектно-ориентированного подхода.

13. Возможности расширения

Проект предусматривает дальнейшее развитие:

добавление новых уровней;

усложнение игровых механик;

добавление новых типов объектов;

расширение визуальных и звуковых эффектов.
